GIT

Git er et versionstyringssystem som kan bruges til at lave git-repositories som man kan dele med andre softwareudviklere. Her i kan man lave branching så man uforstyret og uden risiko for at ødelægge andre funktioner og systemer, kan udvikle sin kode. For at pushe sin kode til sit repository kan man bruge disse kommandoer i terminalen.

  • git add .
  • git add -A (Stager alle ændringer til commit)
  • git commit -m "Beskrivelse af mit commit"
  • git push origin <"branch-navn">

Node JS

Node er et open-source runtime environment som kan bruges til udføre javascript kode. Det bruges ofte til at brugge backenden for en web applikation, i forhold til API'er. Node er meget skalerbart, kan klare meget data intensive situationer og kører real-time backend services. Node er meget hurtigt at sætte op og er rigtig godt til at udarbejde prototyper. Dertil er det hurtigere at udarbejde applikationer i, kræver mindre kode og filer og det har en hurtigere response tid. Dertil har det er meget stort offentligt open-source libary, som gør at du kan genbruge andres kode, så alt ikke skal bygges af en selv. For at køre projekter med node js skal man opsætte et projekt med følgende mapper og filer:

  • node_modules - som hentes ned ved at bruge følgende kommando - npm install express
  • package-lock.json
  • package.json - her henter man express ned som en dependency
  • app.js - hvor man her initialiserer express og opsætter en port som den skal lytte på

Derefter bruge man kommandoen "node app.js" for at køre serveren

REST API

REST API står for, Representational State Transfer Application Programming Interface. Det er en stil til at designe netværksbaserede applikationsgrænseflader. Det er hvad der bliver kaldt når man gerne vil fremvise en specifik side, opdatere/slette/tilføje noget i databasen. Her kigger man på 5 forskellige HTTP metode kald, GET, POST, PUT, PATCH og DELETE.


Når man navngiver sit endpoint giver man det samme navneord og i flertal, da det gør det mere læsbart.

GET metode til at hente drinks ud fra Id


POST metode til at tilføje en drinks til database


PUT metode til at opdatere alle variabler på en drink ud fra et Id


PATCH metode til at opdatere dele af en drink ud fra et Id


DELETE metode til at en drinks fra databasen ud fra et Id


Type coercion and how to avoid it

Coercion er en proces, hvor et sprog automatisk konverterer en værdi fra en datatype til en anden. I JavaScript forekommer coercion ofte i forbindelse med operationer eller sammenligninger, hvor værdier med forskellige datatyper interagerer. Der er 2 typer coercion:

  • Implicit coercion - hvor det forekommer automatisk når javascript konverterer værdier fra en datatype til en anden, uden at blive bedt om det af udvikleren
  • Explicit coercion - Dette er coercion hvor man benytter sig af indbyggede funktioner til at konvertere en værdis datatype til en anden. F.eks. parseInt()
